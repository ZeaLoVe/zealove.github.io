# 星垒防线（StarFort Defense）游戏设计Spec v2.0

## 1. 游戏概述

| 字段 | 说明 |
|------|------|
| 游戏名称 | 星垒防线（StarFort Defense） |
| 游戏类型 | 休闲塔防 + 策略资源管理 |
| 目标平台 | H5（Web） |
| 目标用户 | 18-35岁上班族，碎片化时间玩家 |
| 单局时长 | 3-5分钟/局（持续挑战） |
| 付费模式 | 广告变现（激励视频为主） |

**游戏简介**：
星垒防线是一款融合"方块构建"+"资源管理"+"塔防抵御"的持续策略小游戏。敌人在游戏过程中持续生成并进攻，玩家需要及时放置方块构建防御塔。每个方块只能攻击10次后消失，玩家需要在有限的资源下合理规划布局，抵御持续来袭的敌人，保护底线不被突破。

**核心亮点**：
- **持续挑战模式**：无波次限制，敌人持续生成，难度递增
- **资源管理策略**：方块攻击次数限制，需要合理分配资源
- **即时反馈体验**：深色格子可放置，浅色格子不可放置，直观清晰
- **快速操作流**：点击放置，右键旋转，操作便捷流畅

## 2. 玩法设计

### 2.1 核心玩法

游戏采用**持续挑战模式**，无阶段切换：

```
放置方块 → 防御塔攻击 → 方块消耗 → 敌人持续来袭 → 得分累积 → 游戏继续
```

**构建阶段（持续进行）**：
- 游戏开始时随机生成一个方块
- 玩家点击网格放置方块，右键点击旋转方块
- 每个位置只能放置一个方块（不可堆叠）
- 方块放置后立即转化为防御塔，自动攻击

**防御阶段（持续进行）**：
- 敌人从屏幕右侧持续生成并向左移动
- 防御塔自动攻击范围内的最近敌人
- 每个方块只能攻击10次，攻击次数耗尽后消失
- 敌人被消灭 → 得分；敌人突破底线 → 扣生命值
- 敌人生成速度和强度随时间递增

### 2.2 玩法循环

**核心循环（单局内）**：
```
放置方块 → 资源消耗 → 抵御敌潮 → 得分 → 方块消失 → 重新放置 → 循环
```

**成长循环（多局积累）**：
- 短期：挑战更高分数，存活更长时间
- 中期：解锁新方块形状（预留）
- 长期：登上本地排行榜，冲击最高分

### 2.3 操作方式

| 操作 | PC端 | 移动端 |
|------|------|--------|
| 选择方块 | 系统随机生成 | 系统随机生成 |
| 放置方块 | 鼠标左键点击目标格子 | 点击目标格子 |
| 旋转方块 | 鼠标右键或右下角按钮 | 双指旋转或旋转按钮 |
| 重新开始 | 右上角按钮 | 右上角按钮 |

### 2.4 胜负条件

- **失败条件**：底线生命值归零（生命值初始 3 点，敌人突破扣 1 点）
- **无尽挑战**：无胜利条件，挑战能坚持多长时间和获得多少分数
- **随时重开**：点击右上角重新开始按钮可立即重置游戏

## 3. 可玩性设计要点

### 3.1 上手体验

| 阶段 | 时长 | 目标 |
|------|------|------|
| 前 30 秒 | 自由探索 | 理解方块放置、旋转、攻击机制 |
| 前 1 分钟 | 渐进挑战 | 体验敌人持续生成，资源管理压力 |
| 第 1 局 | 完整体验 | 感受到策略深度和持续挑战乐趣 |

**新手引导设计**：
- 界面提示精简："方块攻击10次 | 深色可放·浅色不可放 | 右键旋转"
- 网格背景颜色区分：深色表示可放置，浅色表示不可放置
- 失败时保留最高分记录，激发再次挑战欲望

### 3.2 目标设定

| 周期 | 目标类型 | 示例 |
|------|----------|------|
| 单局 | 短期 | 存活更长时间，获得更高分数 |
| 每日 | 中期 | 打破昨日最高分记录 |
| 长期 | 长期 | 登上本地排行榜前三 |

### 3.3 反馈系统

**视觉反馈**：
- 放置方块：立即显示防御塔，颜色区分（蓝、红、紫、黄、绿）
- 防御塔透明度：随剩余攻击次数变化（10次最亮，1次最暗）
- 攻击次数显示：防御塔上显示剩余攻击次数数字
- 网格背景：深色可放置，浅色不可放置
- 消灭敌人：敌人消失，得分增加
- 突破底线：屏幕闪烁红色，生命值减少

**数值反馈**：
- 生命值：实时显示当前生命值（初始3点）
- 得分：实时更新，屏幕顶部居中
- 游戏时间：内部计时，控制敌人生成节奏

### 3.4 动力设计

| 类型 | 实现方式 |
|------|----------|
| 本地排行 | 保存最高分记录，显示历史最佳 |
| 快速重开 | 一键重新开始，挑战更高分数 |
| 简洁提示 | 界面文字精简，不干扰游戏体验 |
| 即时反馈 | 每次操作都有明确视觉反馈 |

### 3.5 难度曲线

```
敌人生成间隔： 2s → 1.8s → 1.6s → ... → 0.5s（最快）
敌人血量：    30 → 35 → 40 → ... → 持续增加
敌人速度：    1.0 → 1.1 → 1.2 → ... → 持续增加
```

**设计原则**：
- 初期难度较低，让玩家熟悉机制
- 随时间缓慢递增，避免突然难度断层
- 最终达到极限挑战，考验玩家策略极限

### 3.6 重玩价值

| 机制 | 说明 |
|------|------|
| 无尽模式 | 敌人持续生成，挑战能坚持多久 |
| 策略变化 | 不同布局策略带来不同游戏体验 |
| 本地排行 | 保存最高分，挑战自我极限 |
| 快速重开 | 一键重开，快速开始新挑战 |

## 9. 游戏设计理念

**核心理念**：简单易懂，策略深度，持续挑战

1. **简单易懂**：操作简单（点击放置、右键旋转），规则清晰（攻击10次、保护底线）
2. **策略深度**：资源有限（每次只能攻击10次），需要合理规划布局
3. **持续挑战**：敌人持续生成，难度递增，考验玩家极限策略能力

**设计目标**：3分钟上手，持续可玩，每次挑战都有新体验
